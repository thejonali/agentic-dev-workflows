# Agentic Developer Workflows v0.1.0

v0.1.0 is the initial stable workflow-pack release. It establishes a
provider-neutral core, reproducible adapters for three providers, deterministic
validation, and a local read-only MCP interface.

## Highlights

- Define workflow behavior once under `core/` and render thin Codex, Claude
  Code, and Cursor adapters.
- Validate seven workflows, fifteen agent roles, fourteen command specs, three
  schemas, ninety-three generated provider assets, and local documentation
  links.
- Query, render, validate, and score repository evidence through five bounded,
  read-only MCP tools.
- Reproduce installed provider examples and representative repository guides.
- Build byte-for-byte reproducible release ZIPs with SHA-256 checksums.

## Release Artifacts

Follow the [installation guide](../installation.md) for archive layouts and
provider-specific setup.

## Compatibility

- Python 3.11 or newer is required for the complete validation and MCP path.
- The MCP server depends on `mcp>=1.12,<2` and uses stdio only.
- Provider assets target the current checked-in Codex, Claude Code, and Cursor
  packaging contracts documented in their installation guides.
- The MCP package is installed from the complete archive or source checkout; it
  is not published to PyPI in v0.1.0.

## Verification Evidence

The release is accepted only when all of these commands pass from the repository
root:

```sh
uv run --with 'mcp>=1.12,<2' -- python -B -m unittest discover -s tests -v
python3 -B scripts/validate_workflows.py
python3 -B scripts/generate_provider_assets.py --check
python3 -B scripts/generate_examples.py --check
git diff --check
```

Release archives are then built and inspected with:

```sh
python3 -B scripts/build_release.py --version 0.1.0
unzip -t dist/agentic-dev-workflows-v0.1.0.zip
(cd dist && shasum -a 256 -c SHA256SUMS-v0.1.0.txt)
```

## Known Limitations

- The generic provider adapter and installer CLI are deferred.
- Cursor custom commands remain a provider beta surface.
- Codex custom prompts and Claude Code compatibility commands are secondary to
  their supported skill interfaces.
- Example Python CLI and React repository guides are illustrative profiles, not
  runnable sample applications.
- The MCP server requires access to the extracted library root and does not run
  as a hosted network service.
- Release artifacts are not signed in v0.1.0; verify them with the published
  SHA-256 manifest and the Git tag.

## Upgrade and Rollback

This is the first versioned release, so no migration is required. Pin automated
installation to `v0.1.0`. To roll back a provider installation, restore the
target repository's previous `.agents/`, `.codex/`, `.claude/`, or `.cursor/`
files. To roll back the MCP server, remove its virtual environment and continue
using the previous source checkout; the server does not write migration state.
